## 接口和抽象类的区别

1. 默认的方法实现：
    - 抽象类可以有默认的方法实现；
    - 接口是完全抽象的，根本不存在方法的实现
2. 实现：
    - 抽象类：子类使用extends关键字来继承抽象类。
    如果子类不是抽象类的话，他需要提供抽象类中的所有抽象方法的实现。
    - 子类使用关键字implements来实现接口，它需要提供接口中所有方法的实现。
3. 构造器：
    - 抽象类可以有构造器
    - 接口不能有构造器
4. 与正常JAVA类的区别
    - 除了不能实例化抽象类以外，它与普通JAVA类没有任何区别
    - 接口是完全不同的类型
5. 访问修饰符：
    - 抽象方法中可以有public,protected,private这些修饰符
    - 接口中的默认修饰符为public，不能有其他
6. main方法：
    - 抽象方法中可以有main方法，可运行
    - 没有main方法，不可以运行
7. 成员变量：
    - 抽象类中的变量可以是各种类型
    - 接口中只能是public static final类型的
